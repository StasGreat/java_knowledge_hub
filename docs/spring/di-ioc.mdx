# DI and IoC

## IoC

Inversion of Control means the framework controls object creation and lifecycle instead of the application code doing that everywhere manually.

## DI

Dependency Injection is a way to provide required dependencies to an object from the outside.

## Example

```java
@Service
public class UserService {
    private final UserRepository userRepository;

    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }
}
```

## Why constructor injection is preferred

- dependencies are explicit
- easier to test
- supports immutable fields
- simpler to reason about than field injection
