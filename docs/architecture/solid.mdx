# SOLID

## Definition

SOLID is a set of five object-oriented design principles that help keep code maintainable, extensible, and easier to test.

## The five principles

- Single Responsibility Principle
- Open/Closed Principle
- Liskov Substitution Principle
- Interface Segregation Principle
- Dependency Inversion Principle

## Practical interpretation

SOLID is not about memorizing five names. It is about reducing change impact and making code easier to extend without breaking existing behavior.
